브랜치명
	
main	
역할 :최종 결과물. 항상 실행 가능한 상태의 최신 코드를 유지합니다.	
사용시기 :프로젝트 완료 후 또는 주요 마일스톤 달성 시

develop	
역할 :개발 중심축. 모든 기능 구현과 테스트가 합쳐지는 통합 브랜치입니다.
사용 시기:	설계 완료 후 구현 단계가 시작될 때 생성

docs (또는 design)	
역할:	분석 및 설계. 요구사항 분석서, UI 설계, DB 스키마 등 문서를 관리합니다.
사용 시기: 	현재 단계 (가장 먼저 생성 추천)

feature/
역할 : 기능명	단위 기능 개발. 특정 기능을 만들 때 develop에서 분기합니다
 사용시기 :구현 단계에서 기능별 작업 시

test	
역할 : 검증 및 안정화. 배포 전 버그를 잡고 성능을 확인합니다.
사용 시기 : 구현 완료 후 main 병합 전 단계
