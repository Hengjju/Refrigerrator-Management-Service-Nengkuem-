1. Ingredient (식재료) - ⭐️ 가장 중요!
-> 식재료 클래스는 데이터의 상태가 가장 활발하게 변하는 곳입니다. 특히 사용자가 요리해서 지우는 행위와, 유통기한이 지나서 시스템이 자동으로 지우는 행위는 완전히 분리된 별개의 기능으로 다루어야 완벽한 관리가 가능합니다.

 - changeIngredientStatus() : (작성하신 것 유지) 상태를 ACTIVE, CONSUMED 등으로 변경합니다.

 - updateDetails() : 식재료의 수량(quantity)이나 메모(memo)를 수정할 때 사용합니다. (유스케이스의 'Modify Ingredient Details' 역할)

 - consume() : 사용자가 요리를 완료해서 수동으로 식재료를 소진 처리(삭제)할 때 호출되는 독립적인 메서드입니다.

 - expire() : 유통기한이 지나서 시스템이 자동으로 폐기 처리(삭제)할 때 호출되는 메서드입니다.

 - isExpired() : 오늘 날짜와 Expiredate를 비교해서 "이거 상했어?"라고 물어볼 때 True/False를 반환하는 계산 메서드입니다.

2. Storage (저장소/냉장고)
-> 저장소는 식재료들을 담고 관리하는 '컬렉션(상자)' 역할을 합니다.

 - getStorages() : 유저가 가진 냉장고, 김치냉장고 등의 '저장소 목록'을 리스트 형태로 쫙 가져옵니다.

 - registerIngredient() :새 식재료를 냉장고에 넣습니다.

 - deleteIngredient() : 냉장고에서 식재료를 뺍니다.

 - moveIngredient() : 특정 식재료를 냉장실에서 냉동실로 옮기는 등 다른 Storage로 이동시킵니다. (유스케이스의 'Move Storage Location' 역할)

 - getExpiringIngredients() : 유통기한이 며칠 안 남은 식재료들만 쏙쏙 뽑아서 보여달라고 요청하는 메서드입니다. (알림이나 대시보드에 뿌려줄 때 씁니다)

 - getIngredients() : 냉장고에 있는 식재료들을 리스트 형태로 쫙 보여줍니다.

3. Recipe (레시피)

 - recommend() : 현재 내 냉장고에 있는 식재료들과 이 레시피의 재료를 비교해서 "몇 %나 일치하는지 계산해서 유저에게 추천해줍니다.

4. Ingredient History (기록)
-> 기록은 한 번 생성되면 내용이 바뀌지 않는(불변) 특성이 있습니다.

 - recordEvent() : 등록, 수동 삭제(소비), 자동 삭제(부패) 등의 사건이 발생했을 때 그 내역을 장부에 한 줄 기록하는 메서드입니다.

5. User (사용자)
-> 사용자는 앱에 들어와서 자신의 냉장고를 관리하는 주체입니다.

 - login() : 유스케이스에 있던 로그인 기능입니다.

 - createStorage()  : 새로운 냉장고(예: '김치냉장고')를 추가

 - removeStorage() : 냉장고(예: '김치냉장고')삭제합니다.
